Comandos Linux

O Linux é um sistema operacional de código aberto amplamente utilizado em servidores, data centers e sistemas embarcados. Ele é conhecido por sua estabilidade, segurança e flexibilidade. Diferente do Windows, o Linux é baseado em uma filosofia modular e o controle do sistema é feito principalmente via linha de comando, o que permite automação, manutenção remota e um domínio completo sobre o sistema.

Navegação no Sistema

O sistema de arquivos no Linux segue uma estrutura hierárquica única, começando em / (raiz). Entender comandos de navegação é essencial para localizar e manipular arquivos.

  • pwd — mostra o diretório atual.
  • ls -l — lista arquivos com detalhes como permissões, dono e tamanho.
  • cd <pasta> — muda o diretório atual.
  • cd .. — volta um nível na hierarquia.
  • tree — exibe a estrutura de diretórios em forma de árvore.

Manipulação de Arquivos

Criar, mover e remover arquivos faz parte do uso cotidiano. Esses comandos permitem gerenciar dados com rapidez e precisão.

  • touch arquivo.txt — cria um novo arquivo vazio.
  • cp origem destino — copia arquivos ou diretórios.
  • mv arquivo.txt nova_pasta/ — move ou renomeia arquivos.
  • rm -r pasta/ — remove diretórios recursivamente (com cuidado).
  • cat arquivo.txt — exibe o conteúdo de um arquivo.

Permissões e Segurança

O Linux é multiusuário, e a segurança é gerenciada através de permissões e donos de arquivos. Três grupos de permissão controlam o acesso: dono, grupo e outros.

  • chmod 755 arquivo — altera as permissões de leitura, escrita e execução.
  • chown usuario:grupo arquivo — muda o dono e o grupo do arquivo.
  • sudo — executa comandos administrativos temporariamente.
  • ps aux — exibe todos os processos em execução.
  • kill PID — finaliza processos travados.

Comandos Git

O Git é um sistema de controle de versão distribuído, criado por Linus Torvalds, que permite rastrear mudanças em projetos e colaborar com eficiência. Cada desenvolvedor possui uma cópia completa do repositório, garantindo integridade e independência de rede. É a base de plataformas como GitHub, GitLab e Bitbucket.

Configuração Inicial

Antes de começar, o Git precisa conhecer sua identidade para registrar corretamente os autores dos commits.

  • git config --global user.name "Seu Nome"
  • git config --global user.email seu@email.com
  • git config --list — exibe as configurações atuais.

Versionamento e Histórico

O Git monitora cada alteração feita em arquivos, permitindo criar versões, desfazer erros e comparar estados do projeto.

  • git init — inicializa um novo repositório Git.
  • git add . — adiciona arquivos à área de preparação (staging area).
  • git commit -m "mensagem" — salva uma nova versão (commit) com descrição.
  • git log — lista o histórico de commits.
  • git diff — mostra diferenças entre versões.

Colaboração e Ramificações

O Git permite criar branches (ramificações) para desenvolver novas funcionalidades de forma isolada, sem interferir no código principal.

  • git branch nome — cria uma nova branch.
  • git checkout nome — alterna para uma branch existente.
  • git merge nome — une uma branch com outra.
  • git push origin main — envia alterações ao repositório remoto.
  • git pull origin main — atualiza o repositório local.

Hospedagem Estática

A hospedagem estática é ideal para sites compostos por HTML, CSS e JavaScript puro. Diferente de servidores dinâmicos, onde há processamento em tempo real, as páginas estáticas são rápidas e seguras, entregues diretamente por redes de distribuição de conteúdo (CDNs). Três das plataformas mais conhecidas são:

Vercel

Focada em frameworks modernos como Next.js, React e Svelte. A Vercel oferece deploy automático a cada commit no GitHub, integração com APIs, funções serverless e cache inteligente via Edge Network. É excelente para projetos com foco em performance e escalabilidade.

Netlify

Conhecida pela simplicidade, a Netlify automatiza deploys, configura HTTPS gratuitamente via Let's Encrypt e oferece funções serverless, redirecionamentos e variáveis de ambiente. Ideal para estudantes e desenvolvedores iniciantes.

Render

O Render combina hospedagem estática com suporte a backends em Node, Python e Go. Permite escalonamento automático, integrações Git e domínio personalizado, sendo ótimo para quem planeja expandir um site estático para um aplicativo completo.

DNS e HTTPS

DNS — Sistema de Nomes de Domínio

O DNS (Domain Name System) é o serviço responsável por traduzir nomes de domínio legíveis por humanos, como www.exemplo.com, em endereços IP, como 192.168.0.1. Ele funciona como uma “agenda telefônica” da Internet. Quando você acessa um site, seu computador pergunta a um servidor DNS qual IP corresponde ao domínio, e então estabelece a conexão com o servidor certo.

O sistema DNS é hierárquico e distribuído, composto por servidores raiz, TLD (Top-Level Domain) e autoritativos. Cada um deles tem funções específicas e armazena informações em registros chamados Resource Records:

  • A — associa um domínio a um endereço IPv4.
  • AAAA — associa um domínio a um endereço IPv6.
  • CNAME — cria um apelido que aponta para outro domínio.
  • MX — indica os servidores responsáveis por e-mails do domínio.
  • NS — define os servidores autoritativos da zona.
  • TXT — armazena textos, usados em SPF e verificações de domínio.
  • CAA — especifica quais autoridades certificadoras podem emitir certificados SSL/TLS.

Consultas DNS podem ser recursivas (onde um servidor busca a resposta completa) ou iterativas (onde ele apenas indica o próximo servidor a consultar). O parâmetro TTL (Time To Live) define por quanto tempo uma resposta pode ser armazenada em cache, afetando diretamente a velocidade de propagação de mudanças.

HTTPS — Segurança e Criptografia

HTTPS (HyperText Transfer Protocol Secure) é a versão segura do HTTP. Ele protege a comunicação entre o navegador e o servidor usando o protocolo TLS (Transport Layer Security). O TLS garante três princípios fundamentais:

  • Confidencialidade — os dados são criptografados e não podem ser lidos por terceiros.
  • Integridade — impede que o conteúdo seja alterado durante o tráfego.
  • Autenticação — o servidor prova sua identidade através de certificados digitais emitidos por Autoridades Certificadoras (CAs).

Durante o Handshake TLS, cliente e servidor trocam informações para negociar a cifra e gerar chaves de sessão. O navegador valida o certificado, verifica sua assinatura e confirma se o domínio é confiável. Uma vez estabelecida a conexão, os dados são transmitidos de forma criptografada.

Hoje, o HTTPS é essencial: protege credenciais, impede ataques Man-in-the-Middle e é exigido por navegadores modernos e mecanismos de busca. Cabeçalhos como HSTS (Strict Transport Security) reforçam a segurança ao obrigar o uso de HTTPS nas próximas conexões.

Plataformas Git

Plataformas baseadas em Git fornecem não apenas hospedagem de repositórios, mas também ferramentas de colaboração, controle de acesso, pipelines CI/CD e integração com sistemas externos. Três das mais conhecidas são:

GitHub

Focado na comunidade open source, o GitHub é o maior ecossistema de desenvolvedores do mundo. Oferece issues, pull requests, GitHub Actions (para automação) e GitHub Pages (para hospedagem gratuita de sites estáticos). É ideal tanto para projetos pessoais quanto corporativos.

GitLab

O GitLab combina versionamento com um conjunto completo de DevOps: CI/CD, pipelines, controle de qualidade e monitoramento. Pode ser usado na nuvem ou instalado em servidores próprios (self-hosted), oferecendo mais privacidade e controle.

Bitbucket

Voltado a equipes empresariais, o Bitbucket pertence à Atlassian e integra-se ao Jira e Confluence. Suporta repositórios privados ilimitados e é amplamente usado em empresas que buscam controle de acesso detalhado e integração com ferramentas corporativas.
